Educational Technologies of 21st Century

SCORM

SCORM is a set of technical standards for E-learning software products and applications. SCORM tells programmers how to write their code so that it can “play well” with other e-learning software. In fact, it is an industry standard for e-learning interoperability. Specifically, SCORM standards govern how online learning content and Learning Management Systems (LMSs) communicate with each other. SCORM is not concerned with instructional design or any other pedagogical interest — it is merely a technical standard. SCORM stands for “Sharable Content Object Reference Model.“ If an LMS is conforming to SCORM standards, it can play any content that is conforming to SCORM, and any SCORM “conformant“content can play in any SCORM “conformant“ LMS. (scorm.com)



SCORM Cloud

SCORM Cloud is another technology that contains fewer features than an LMS. SCORM Cloud mostly can delivers the content to the learner and tracks the results. SCORM Cloud is a type of technology that many organizations prefer to use while others, in educationfor instance, need the LMS features. Unlike LMS these are some features that SCORM Cloud fails to offer:

  • “Competency Management
  • Talent management
  • Manage users, roles, instructors, facilities
  • Enrollment approval workflow
  • e-Commerce for course enrollments
  • Skill gap analysis
  • Tracking compliance requirements”
(scorm.com)

Tin Can API

While SCORM has only been around for a decade, it has already been considered an old technology. The Tin Can API (Sometimes known as the Experience API, or xAPI) is a brand new development in learning technology. Tin Can API makes it possible to collect data about a wide range of experiences an individual has (online and offline). Tin Can API could capture data in a consistent format about a person or group’s activities from many technologies. Using Tin Can's simple vocabulary, very different systems can securely communicate by capturing and sharing this stream of activities. Compared to the former 'specifications' which were challenging and had some limitations; Tin Can API is simple and flexible. “Mobile learning, simulations, virtual worlds, serious games, real-world activities, experiential learning, social learning, offline learning, and collaborative learning” are the learning technologies that can now be recognized and communicated effectively by the presence of the Tin Can API. These are the steps that explain how the Tin Can API works: Learning occurs through people's interactions, and contents. These actions can happen anywhere and signal an event where learning could happen. Tin Can API could record all of these interactions. When there is a need for an activity to be recorded, the application sends secure statements in the form of “Noun, verb, object” or “I did this” to a Learning Record Store(LRS); an LRS can exist independently on its own, or inside an LMS. At the final step, Learning Record Stores records all the statements that have been made.The Tin Can API is community-driven so that LRS can share the information with other LRSs. (tincanapi.com)


Semantic interoperability

The latest technology in learning technology is Semantic interoperability. “Semantic interoperability is when information—the meaning behind captured data—is portable and well understood by any subsequent system requesting and reviewing it”. (tincanapiblog.com)
